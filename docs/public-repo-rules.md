# Public Repository Rules

Use these rules before pushing this workspace to a public repository.

## Visibility

- Do not commit private client files, signed contracts, credentials, bank data, or personally identifying information.
- Keep client testimonials public only when written permission exists.
- Replace unverified success metrics with placeholders until source data is confirmed.

## Branch Rules

- Protect `main` from deletion and force pushes.
- Require pull requests for changes to `main`.
- Require one approving review for public-facing copy, legal/compliance material, and grant facts.
- Require conversation resolution before merge.

## Quality Rules

- Verify source links before updating grant opportunities.
- Include source notes in pull requests when dates, funding amounts, eligibility, or agency names change.
- Keep one grant research topic per folder and one primary Markdown file per topic.
- Keep website changes free of private client information.

## Suggested Labels

- `grant-research`
- `website`
- `showcase`
- `legal-compliance`
- `financials`
- `needs-source`
- `expired-opportunity`
